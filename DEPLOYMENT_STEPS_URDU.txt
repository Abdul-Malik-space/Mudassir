DashBoard2 Vercel Deployment Steps

1) GitHub repo me ye clean project upload karein.
   Important: node_modules, dist, .env aur .git upload na karein.

2) Backend project deploy karein:
   - Vercel > Add New Project
   - Same GitHub repo select karein
   - Root Directory: backend
   - Environment Variables:
     MONGO_URI = your MongoDB Atlas URI
     CLIENT_URL = *
   - Deploy
   - Test URL: https://your-backend.vercel.app/api/health

3) Frontend project deploy karein:
   - Vercel > Add New Project
   - Same GitHub repo select karein
   - Root Directory: frontend
   - Framework Preset: Vite
   - Build Command: npm run build
   - Output Directory: dist
   - Environment Variables:
     VITE_API_URL = https://your-backend.vercel.app/api
   - Deploy

4) Agar kisi aur banday ko link open nahi ho raha:
   - Vercel dashboard > Project > Settings > Deployment Protection
   - Protection / Vercel Authentication ko OFF karein
   - Project public hona chahiye
   - Backend ka CLIENT_URL ab * rakha gaya hai taake CORS access block na kare

5) MongoDB Atlas:
   - Network Access me IP allow karein.
   - Testing ke liye 0.0.0.0/0 use kar sakte hain.
   - Production ke liye restricted access better hai.
